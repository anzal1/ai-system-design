# Launch Plan

The goal is to attract serious builders, not generic AI traffic.

## Launch Criteria

Launch publicly when the repo has:

- Clear README positioning
- Course and syllabus
- At least 10 strong pages
- At least 3 runnable labs
- At least 3 case studies
- Contribution rules
- Source map
- Issue templates

This repo now meets that bar.

## Launch Message

```text
I am building AI System Design, an open-source course and handbook for engineers designing production AI systems.

It covers RAG, agents, evals, observability, model routing, MCP/tool gateways, security, latency, cost, and real architecture tradeoffs.

The goal is not AI news or prompt hacks. It is production system design: diagrams, failure modes, eval plans, labs, and design reviews.

I would love sharp critique from engineers building AI systems.
```

## What To Ask For

Ask for critique, not stars.

Good prompts:

- Which architecture topics are missing?
- Which claims need better sources?
- Which labs would make this more useful?
- Which production failure modes have you seen?
- Which case studies should be added next?

## Success Signals

Stronger than stars:

- Engineers opening detailed issues
- Corrections from practitioners
- Requests for specific patterns
- PRs adding failure modes or evals
- People referencing pages in design discussions
- Companies using assignments internally
