# Design Reviews

Design reviews are worked examples of AI system architecture.

They should show how a strong engineer evaluates a system: requirements, constraints, architecture, tradeoffs, failure modes, evals, observability, security, and rollout.

## Review Structure

Each design review should include:

- Scenario
- Requirements
- Architecture
- Data flow
- Design decisions
- Alternatives considered
- Failure modes
- Evaluation plan
- Observability plan
- Security review
- Cost and latency budget
- Rollout plan
- Open questions
